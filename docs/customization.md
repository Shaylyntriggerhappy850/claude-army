# Customization

How to modify agents generated by the forge.

## Adjusting Generated Agents

The forge creates agents in `~/.claude/agents/`. They're markdown files — edit them directly.

### Adding Checks to the Reviewer

Open `~/.claude/agents/<project>-reviewer.md` and add checks under the appropriate section:

```markdown
### Step N: Custom Check
- [ ] Verify that [condition]
- [ ] Ensure [requirement]
```

### Updating the Challenger After a Real Review

When the lead reviewer comments on your PR, feed their new patterns back:

1. Read their comments
2. Add new patterns to the appropriate tier in the challenger agent
3. If they approved something you thought was wrong, add it to "Things NOT to flag"

### Changing the Pipeline Order

Edit the orchestrator's phases. You can:
- Skip phases (remove them)
- Add phases (insert new agent calls)
- Change checkpoint locations (move the ⏸ markers)

## Creating Additional Agents

Use the forge's template agents in `templates/` as starting points. Copy one, fill in the `{{placeholders}}` with your project's specifics.

## Multiple Projects

The forge creates agents prefixed with the project name (`<project>-reviewer`), so agents from different projects don't conflict. You can have agent suites for multiple projects simultaneously in `~/.claude/agents/`.
